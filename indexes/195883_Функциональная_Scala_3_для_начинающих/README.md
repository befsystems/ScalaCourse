# Индексы курса

Эта директория содержит автоматически генерируемые индексные файлы для навигации по контенту курса Scala.

## Назначение

Индексы предназначены для:
- Быстрой навигации LLM по материалам курса
- Поиска уроков по темам и ключевым словам
- Понимания структуры и организации курса
- Определения типов контента (теория/практика/тесты)

## Структура индексов

### Генерируемые файлы

1. **module-hierarchy.md** - Иерархическая структура
   - Древовидное представление: курс → модули → уроки → шаги
   - Подходит для понимания общей организации курса
   - Показывает количество шагов в каждом уроке

2. **topics-keywords.md** - Тематический индекс
   - Группировка по темам и ключевым словам
   - Позволяет найти материалы по конкретной теме (например, "val", "pattern matching")
   - Подходит для быстрого поиска релевантного контента

3. **quick-reference.md** - Табличная навигация
   - Сжатая табличная форма
   - Одна строка = один урок
   - Быстрый обзор всего курса

4. **by-content-type.md** - Группировка по типам
   - Раздельные секции для теории, практики, тестов
   - Полезно для поиска материалов определённого типа
   - Типы: text (теория), code (практика), choice (тесты с выбором), matching (задания на соответствие)

## Генерация индексов

Для (пере)генерации индексов используй команду:

```
@index-course
```

**Как это работает:**
- Команда автоматически запускает отдельного агента
- Агент обрабатывает весь курс поэтапно (по 2-3 модуля за итерацию)
- Агент создаёт все 4 типа индексов и обновляет этот README
- Процесс выполняется в изолированном контексте, не перегружая основную сессию

**Альтернатива (прямой запуск агента):**
```
Используй промпт из prompts/commands/index-course-agent.md
```

## Принципы организации

- **Компактность**: Индексы оптимизированы по размеру для загрузки в контекст LLM
- **Многомерность**: Разные индексы для разных типов навигации
- **Актуальность**: Индексы генерируются на основе реального контента
- **Относительные пути**: Все пути относительно корня проекта

## Обслуживание

- Пересоздавай индексы после добавления/изменения контента курса
- Проверяй корректность путей после реорганизации структуры
- Индексы являются производными данными - не редактируй их вручную

## Возобновление прерванной индексации

Если процесс индексации был прерван:

1. Запусти команду `@index-course` снова
2. Агент обнаружит файл `.indexing-state.json` в этой директории
3. Агент предложит:
   - **Возобновить** - продолжить с места остановки (рекомендуется)
   - **Начать заново** - удалить прогресс и начать сначала

**Файл состояния:** `.indexing-state.json`
- Временный технический файл
- Создаётся автоматически при индексации
- Сохраняет прогресс после каждой итерации
- Удаляется автоматически после успешного завершения
- НЕ коммитится в git (добавлен в `.gitignore`)

**Безопасность:**
- Можно вручную удалить `.indexing-state.json` чтобы начать заново
- Не удаляй этот файл если хочешь возобновить прерванную индексацию
