# Индексы курса "Система типов Scala 3"

> Навигационная система для эффективного изучения и поиска материалов курса

---

## Назначение

Эти индексы предоставляют множественные точки входа для навигации по материалам курса "Система типов Scala 3" (ID: 269026). Они оптимизированы для использования LLM-агентами и людьми, обеспечивая быстрый поиск нужных материалов по разным критериям.

---

## Структура индексов

Система индексации включает 4 типа навигационных файлов:

### 1. module-hierarchy.md - Иерархическая навигация

**Назначение:** Древовидная структура курса для последовательного изучения

**Используйте когда:**
- Изучаете курс последовательно, модуль за модулем
- Нужен обзор структуры всего курса
- Ищете материалы по конкретному уроку

**Организация:** Модули → Уроки → Шаги с детальной информацией о каждом шаге

### 2. topics-keywords.md - Тематический указатель

**Назначение:** Алфавитный поиск по ключевым словам и темам

**Используйте когда:**
- Ищете материалы по конкретной теме (например, "Option", "ADT", "кардинальность")
- Нужно найти все упоминания термина в курсе
- Изучаете связанные концепции

**Организация:** Алфавитный список терминов с ссылками на все упоминания

### 3. quick-reference.md - Табличная навигация

**Назначение:** Компактный обзор всех материалов курса

**Используйте когда:**
- Нужен быстрый обзор всего курса
- Ищете материал по ID или позиции
- Оцениваете объем и распределение контента

**Организация:** Таблицы с кратким описанием каждого шага

### 4. by-content-type.md - Группировка по типам контента

**Назначение:** Фильтрация по типу учебного материала

**Используйте когда:**
- Хотите изучать только теорию (text)
- Ищете практические задания (code)
- Нужны тесты для самопроверки (matching)

**Организация:** Группировка всех шагов по типам: text, code, matching

## Генерация индексов

Индексы генерируются автоматически с помощью системы промптов:

```bash
# Запуск индексации для курса 269026
Используй промпт из файла prompts/commands/index-course-coordinator.md

Курс: content/269026__Система_типов_Scala_3/
Индексы: indexes/269026_Система_типов_Scala_3/
```

**Процесс индексации:**
1. Координатор анализирует структуру курса и планирует итерации
2. Агенты-обработчики извлекают метаданные из файлов курса
3. Агент-генератор создаёт 4 типа индексных файлов
4. Генерируется README.md с финальной статистикой

## Принципы организации

### Многомерная навигация

Каждый тип индекса предоставляет свой "вид" на один и тот же контент:
- **Иерархия** - структурный взгляд (как организован курс)
- **Темы** - семантический взгляд (о чем курс)
- **Таблица** - обзорный взгляд (что есть в курсе)
- **Типы** - педагогический взгляд (как изучать курс)

### Оптимизация для LLM

Индексы спроектированы для эффективной работы с LLM:
- Компактное представление метаданных
- Относительные пути от корня проекта
- Структурированный Markdown для легкого парсинга
- Контекстная информация в каждом индексе

### Поддержка кириллицы

Все индексы полностью поддерживают русский язык:
- Тематический указатель с разделением кириллицы и латиницы
- UTF-8 кодирование всех файлов
- Корректная алфавитная сортировка

## Обслуживание индексов

### Когда обновлять индексы

Индексы требуют обновления при:
- Добавлении новых модулей, уроков или шагов
- Изменении структуры курса (переименование, перемещение)
- Существенных изменениях в содержании (новые темы, ключевые слова)

### Как обновить индексы

1. Запустите процесс индексации через координатора
2. Система автоматически:
   - Обнаружит все изменения в структуре курса
   - Извлечет обновленные метаданные
   - Пересоздаст все 4 индексных файла
   - Обновит статистику в README.md

**Примечание:** Индексация всегда пересоздает индексы полностью (не инкрементально) для обеспечения консистентности данных.

## Примеры использования

### Пример 1: Найти все материалы про Option

1. Откройте `topics-keywords.md`
2. Найдите секцию "О" → "Option"
3. Получите список всех шагов, где упоминается Option

### Пример 2: Изучить практические задания

1. Откройте `by-content-type.md`
2. Перейдите в секцию "code - Практические задания"
3. Выберите интересующий урок

### Пример 3: Обзор структуры модуля

1. Откройте `module-hierarchy.md`
2. Найдите нужный модуль (например, "1_Типы_данных")
3. Просмотрите все уроки и их содержание

### Пример 4: Быстрый поиск по ID

1. Откройте `quick-reference.md`
2. Используйте поиск браузера (Ctrl+F) для поиска по ID шага
3. Получите полную информацию о шаге в табличном формате

## Технические детали

### Формат путей

Все пути в индексах:
- Относительные от корня проекта
- Используют прямые слэши `/`
- Формат: `content/269026__Система_типов_Scala_3/{module}/{lesson}/{step}.md`

### Типы контента

- `text` - теоретический материал, лекции
- `code` - практические задания с кодом
- `matching` - тесты на соответствие
- `choice` - тесты с выбором ответа
- `fill-blanks` - задания на заполнение пропусков

### Метаданные индексов

Каждый индексный файл содержит метаданные в HTML-комментарии:
```markdown
<!--
Сгенерировано: YYYY-MM-DD HH:MM:SS
Модулей: X
Уроков: Y
Шагов: Z
-->
```
