# Структура результатов исследований

## Назначение

Этот каталог содержит результаты исследовательских задач: анализ подходов, изучение технологий, эксперименты, сравнения альтернатив.

## Структура

```
research/
├── topics/                # Исследования тем курса
│   ├── collections/       # Исследование коллекций Scala
│   ├── concurrency/       # Исследование многопоточности
│   └── functional/        # Исследование ФП концепций
│
├── pedagogy/              # Педагогические исследования
│   ├── methods/           # Методы обучения
│   ├── exercises/         # Типы упражнений
│   └── progression/       # Прогрессия сложности
│
├── technical/             # Технические исследования
│   ├── tools/             # Инструменты и платформы
│   ├── integrations/      # Интеграции
│   └── automation/        # Автоматизация процессов
│
├── competitors/           # Анализ конкурентов
│   └── courses/           # Другие курсы Scala
│
├── archive/               # Архив завершенных исследований
│   └── 2025/
│       └── 12/
│
└── STRUCTURE.md           # Этот файл
```

[Что нужно выяснить, какой вопрос ответить]

## Типы исследований

### Технические исследования
- Изучение возможностей Scala 3
- Сравнение библиотек и фреймворков
- Анализ производительности
- Исследование инструментов разработки

### Педагогические исследования
- Анализ методов обучения программированию
- Изучение прогрессии сложности
- Исследование типов упражнений
- Анализ обратной связи студентов

### Контентные исследования
- Анализ тем для курса
- Изучение существующих курсов
- Исследование best practices
- Анализ актуальности материалов

## Жизненный цикл исследования

```
1. Создание задачи (tasks/planning/research-*.md)
   ↓
2. Проведение исследования
   ↓
3. Документирование результатов (research/*/[topic].md)
   ↓
4. Принятие решения
   ↓
5. Создание плана/задач на основе решения
   ↓
6. Архивирование (research/archive/)
```

## Принципы работы

### ✅ DO

- Создавай исследование для каждой задачи типа `research-*`
- Документируй все рассмотренные варианты
- Указывай источники информации
- Делай конкретные выводы и рекомендации
- Архивируй завершенные исследования

### ❌ DON'T

- Не создавай исследования без связанной задачи
- Не оставляй исследования без выводов
- Не принимай решения без анализа альтернатив
- Не забывай документировать эксперименты

## Связь с задачами планирования

Каждое исследование создается для задачи типа `tasks/planning/research-*.md`:

**Задача** → определяет что исследовать
**Исследование** → документирует процесс и результаты
**План** → создается на основе решения из исследования

## Примеры

### Пример 1: Техническое исследование

`research/topics/collections/immutable-vs-mutable.md`:
```markdown
# Исследование: Immutable vs Mutable коллекции в обучении

## Цель
Определить оптимальный подход к представлению коллекций в курсе

## Исследуемые варианты
1. Сначала immutable, потом mutable
2. Сначала mutable, потом immutable
3. Параллельное изучение

## Выводы
Начинающим проще mutable, но для Scala 3 важнее immutable

## Рекомендации
Начать с immutable, показать mutable как альтернативу
```

### Пример 2: Педагогическое исследование

`research/pedagogy/exercises/code-vs-quiz.md`:
```markdown
# Исследование: Соотношение кодовых упражнений и квизов

## Цель
Определить оптимальное соотношение типов упражнений

## Результаты
Анализ 5 популярных курсов показал 70% код / 30% квизы

## Рекомендации
Использовать 3 кодовых упражнения на 1 квиз
```
