# Структура бизнес-требований

## Назначение

Этот каталог содержит внешние бизнес-требования к курсу от заказчиков, платформы Stepik, и других стейкхолдеров.

## Структура

```
requirements/
├── stakeholders/           # Требования от стейкхолдеров
│   ├── platform/          # Требования платформы Stepik
│   ├── audience/          # Требования целевой аудитории
│   └── sponsors/          # Требования спонсоров/партнеров
│
├── course/                # Требования к курсу
│   ├── structure.md       # Требования к структуре курса
│   ├── content.md         # Требования к контенту
│   ├── exercises.md       # Требования к упражнениям
│   └── assessment.md      # Требования к оценке знаний
│
├── technical/             # Технические требования
│   ├── platform.md        # Требования платформы
│   ├── tools.md           # Требуемые инструменты
│   └── compatibility.md   # Требования совместимости
│
└── STRUCTURE.md           # Этот файл
```

## Формат документа требований

```markdown
# Требование: [Название]

## Источник
- **От кого**: [Стейкхолдер]
- **Дата**: [YYYY-MM-DD]
- **Приоритет**: [critical/high/medium/low]
- **Статус**: [new/accepted/in-progress/completed/rejected]

## Описание
[Детальное описание требования]

## Обоснование
[Почему это требование важно]

## Критерии приемки
- [ ] Критерий 1
- [ ] Критерий 2
- [ ] Критерий 3

## Влияние на проект
[Какие части проекта затрагивает]

## Связи
- План: [ссылка на plans/]
- Задачи: [ссылки на tasks/]

## История изменений
- YYYY-MM-DD: Требование получено
- YYYY-MM-DD: Требование принято к реализации
```

## Принципы работы

### ✅ DO

- Документируй все внешние требования
- Указывай источник и дату получения
- Оценивай приоритет и влияние на проект
- Связывай требования с планами и задачами
- Обновляй статус по мере реализации

### ❌ DON'T

- Не создавай внутренние требования (используй plans/)
- Не дублируй информацию из других файлов
- Не принимай требования без анализа влияния

## Связь с другими компонентами

```
requirements/    (внешние требования)
   ↓
plans/          (планы реализации)
   ↓
tasks/          (задачи выполнения)
   ↓
content/        (результат)
```
