# Сгенерированные составные промпты

Эта директория содержит промпты, созданные путем композиции модулей из `modules/` по шаблонам из `templates/`.

## Структура

```
composed/
├── [topic-name]/       # Промпты для конкретной темы
│   ├── v1.md           # Первая версия
│   ├── v2.md           # Улучшенная версия
│   └── final.md        # Финальная версия
└── README.md           # Этот файл
```

## Формат имени папки

`[topic-name]` - название темы урока в kebab-case, например:
- `pattern-matching`
- `option-types`
- `collections-intro`

## Версионирование

- **v1.md** - первая версия промпта
- **v2.md, v3.md...** - итеративные улучшения
- **final.md** - финальная версия, которая использовалась для создания материала

## Зачем хранить версии

1. **История эволюции** - видно как промпт улучшался
2. **Обучение** - можно понять что работает, что нет
3. **Переиспользование** - можно взять рабочий промпт для похожей темы
4. **Документация** - фиксация того, что использовалось

## Использование

Чтобы создать новый составной промпт:

```
@compose-prompt:
  шаблон: templates/lesson-basic.md
  тема: "Pattern Matching в Scala 3"
  уровень: начальный
  сохранить: composed/pattern-matching/v1.md
```

Claude:
1. Прочитает шаблон `lesson-basic.md`
2. Загрузит все указанные модули
3. Применит специфику (тема, уровень)
4. Создаст финальный промпт
5. Сохранит в `composed/pattern-matching/v1.md`

## Лучшие практики

### ✅ DO
- Создавай отдельную папку для каждой темы
- Версионируй промпты (v1, v2, v3)
- Помечай финальную версию как `final.md`
- Коммить изменения в Git с описанием

### ❌ DON'T
- Не удаляй старые версии
- Не смешивай темы в одной папке
- Не редактируй старые версии (создавай новые)
