# Модуль: Инструкции по работе с промптами

## Назначение
Содержит пошаговые инструкции и процедуры для работы с модульной системой промптов.

## Применимость
- Тип промпта: команды/роли (prompt-create)
- Обязательный: да (для роли создания промптов)
- Комбинируется с: roles/prompt-creation.md

## Содержание

### Контекст проекта

Проект использует модульную систему промптов:
- **prompts/modules/** - переиспользуемые блоки
- **prompts/templates/** - готовые шаблоны
- **prompts/commands/** - роли и команды
- **prompts/composed/** - сгенерированные промпты

### Основные операции

#### 1. Создание нового модуля

**Процесс:**
1. Определи категорию (common/, lesson/, exercise/, pedagogy/, roles/, instructions/)
2. Используй шаблон из `prompts/STRUCTURE.md` (строки 173-200)
3. Создай файл в соответствующей подпапке
4. Документируй назначение, применимость и примеры

**Пример:**
```markdown
Создай модуль для интерактивных элементов урока:
- Категория: lesson/
- Название: interactive-elements.md
- Назначение: требования к интерактивным частям
```

#### 2. Композиция промпта из модулей

**Процесс:**
1. Выбери базовый шаблон из `prompts/templates/`
2. Определи дополнительные модули
3. Скомпонуй финальный промпт
4. Сохрани в `prompts/composed/[тема]/`

**Пример:**
```markdown
Создай промпт для урока про Pattern Matching:
- Шаблон: templates/lesson-basic.md
- Дополнительно: modules/pedagogy/motivation.md
- Сохранить: composed/pattern-matching/v1.md
```

#### 3. Редактирование существующих промптов

**Процесс:**
1. Прочитай существующий файл
2. Определи что нужно изменить
3. Сохрани изменения
4. Обнови версию если необходимо

#### 4. Добавление команды в quick-commands

**Процесс:**
1. Открой `prompts/commands/quick-commands.md`
2. Добавь команду в соответствующую секцию
3. Опционально создай файл активации для роли

### Структура и правила

**Детальная документация:**
- Система промптов: `prompts/STRUCTURE.md`
- Быстрые команды: `prompts/commands/quick-commands.md`

**Принципы:**
- Модули атомарные и самодостаточные
- Шаблоны = композиции модулей
- Версионирование в Git
- DRY - переиспользование блоков

### Примеры запросов

```
"Создай модуль для работы с видео-материалами"
"Скомпонуй промпт для продвинутого урока про ADT"
"Добавь команду @stepik-export в quick-commands"
"Отредактируй модуль common/code-style.md"
```

### Инструменты

Используй стандартные инструменты:
- **Read** - чтение существующих файлов
- **Write** - создание новых файлов
- **Edit** - редактирование существующих
- **Glob/Grep** - поиск по структуре
