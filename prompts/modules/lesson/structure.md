# Модуль: Структура урока

## Назначение
Определяет стандартную структуру урока для курса Scala.

## Применимость
- Тип промпта: уроки
- Обязательный: да
- Комбинируется с: common/*, pedagogy/*

## Содержание

### Стандартная структура урока

Каждый урок состоит из следующих секций:

#### 1. Введение (2-3 минуты)
- **Цель**: Зацепить интерес, показать зачем это нужно
- **Содержание**:
  - Краткое описание темы
  - Практическая мотивация (где используется)
  - Связь с предыдущими уроками
  - Что студент узнает

```markdown
# Pattern Matching в Scala

Pattern matching - это мощный инструмент Scala для работы с данными.
Вы уже встречали его при работе с case классами, теперь узнаете все возможности.

После этого урока вы сможете:
- Использовать pattern matching с различными типами данных
- Применять guards и pattern binders
- Понимать exhaustiveness checking
```

#### 2. Теоретическая часть (10-15 минут)
- **Цель**: Объяснить концепцию
- **Содержание**:
  - Определение и синтаксис
  - Ключевые концепции
  - Как это работает под капотом (если важно)
  - Сравнение с альтернативами (если есть)

**Принципы изложения**:
- От простого к сложному
- Один концепт - один параграф
- Визуализация через примеры

#### 3. Примеры кода (8-12 минут)
- **Цель**: Показать применение на практике
- **Количество**: 3-5 примеров нарастающей сложности
- **Структура каждого примера**:
  1. Описание задачи
  2. Код с комментариями
  3. Объяснение ключевых моментов
  4. Вывод/результат

```scala
// Пример 1: Базовое использование
val day = 3
val dayName = day match
  case 1 => "Понедельник"
  case 2 => "Вторник"
  case 3 => "Среда"
  // ...

// Что здесь происходит:
// 1. Проверяем значение переменной day
// 2. Для каждого случая возвращаем строку
// 3. Результат присваивается в dayName
```

#### 4. Распространенные ошибки (3-5 минут)
- **Цель**: Предупредить о типичных проблемах
- **Содержание**:
  - 2-3 частые ошибки новичков
  - Как их избежать
  - Правильный подход

```scala
// ❌ Ошибка: забыли case для всех вариантов
val result = value match
  case 1 => "one"
  case 2 => "two"
  // Компилятор предупредит: match may not be exhaustive

// ✅ Правильно: используем default case
val result = value match
  case 1 => "one"
  case 2 => "two"
  case _ => "other"
```

#### 5. Лучшие практики (2-3 минуты)
- **Цель**: Показать идиоматичный подход
- **Содержание**:
  - Рекомендации по использованию
  - Когда применять, когда нет
  - Связь с другими возможностями языка

#### 6. Практическое задание (5-10 минут)
- **Цель**: Закрепить знания
- **Содержание**:
  - Описание задачи
  - Входные данные
  - Ожидаемый результат
  - Подсказки (опционально)
  - Автотесты

#### 7. Резюме (1-2 минуты)
- **Цель**: Подвести итоги
- **Содержание**:
  - Ключевые выводы (3-5 пунктов)
  - Ссылка на следующий урок
  - Дополнительные материалы (опционально)

### Требования к объему

- **Общий объем**: 30-45 минут на прохождение
- **Текст**: 800-1500 слов
- **Примеры кода**: 3-5 штук
- **Упражнения**: 1-2 задачи

### Тон и стиль

- **Обращение**: на "вы"
- **Стиль**: дружелюбный, но профессиональный
- **Сложность языка**: доступный для аудитории
- **Использование терминов**: с объяснением при первом упоминании

## Примеры использования

В промпте для создания урока:
```
Создай урок следуя стандартной структуре:
1. Введение с мотивацией (2-3 мин)
2. Теоретическая часть (10-15 мин)
3. Примеры кода (3-5 штук, нарастающая сложность)
4. Распространенные ошибки (2-3 примера)
5. Лучшие практики
6. Практическое задание с тестами
7. Резюме

Общий объем: 30-45 минут, 800-1500 слов.
```
