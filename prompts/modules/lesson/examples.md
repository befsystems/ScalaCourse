# Модуль: Требования к примерам

## Назначение
Определяет требования к примерам кода в уроках.

## Применимость
- Тип промпта: уроки
- Обязательный: да (для уроков с кодом)
- Комбинируется с: lesson/structure, common/code-style

## Содержание

### Принципы создания примеров

#### 1. Прогрессивная сложность
Примеры должны идти от простого к сложному:
- **Пример 1**: Базовый синтаксис, минимальный код
- **Пример 2**: Добавление одной новой возможности
- **Пример 3**: Комбинирование возможностей
- **Пример 4-5**: Реалистичные use cases

#### 2. Практическая ценность
Каждый пример должен:
- Решать реальную задачу
- Быть понятным без контекста
- Демонстрировать конкретную возможность

#### 3. Полнота
Примеры должны:
- Компилироваться и запускаться
- Включать необходимые импорты
- Показывать вывод/результат

### Структура примера

```markdown
## Пример [N]: [Название - что демонстрирует]

[Краткое описание задачи и контекста]

```scala
// Код с комментариями
[код]
```

**Что здесь происходит:**

2. [Пояснение шага 2]

```
[вывод программы]
```

### Примеры по типам

#### Базовый пример (Пример 1)
```scala
// Демонстрация базового синтаксиса
val numbers = List(1, 2, 3, 4, 5)
val doubled = numbers.map(n => n * 2)
println(doubled)  // List(2, 4, 6, 8, 10)
```

Характеристики:
- Минимальный код (3-5 строк)
- Одна концепция
- Очевидный результат

#### Практический пример (Пример 3-4)
```scala
// Реальная задача: фильтрация и преобразование пользователей
case class User(name: String, age: Int, isActive: Boolean)

val users = List(
  User("Alice", 25, true),
  User("Bob", 17, true),
  User("Charlie", 30, false)
)

// Получаем имена активных взрослых пользователей
val activeAdults = users
  .filter(u => u.isActive && u.age >= 18)
  .map(_.name)

println(activeAdults)  // List(Alice)
```

Характеристики:
- Реалистичный сценарий
- Комбинирование операций
- Понятная бизнес-логика

### Антипаттерны

#### ❌ Плохой пример
```scala
// Слишком сложно для начального примера
def f[A, B](xs: List[A])(fn: A => B): List[B] =
  xs.foldRight(List.empty[B])((a, acc) => fn(a) :: acc)
```

Проблемы:
- Сразу дженерики
- Непонятные имена (xs, fn, a, acc)
- Сложная концепция (fold)

#### ✅ Хороший пример
```scala
// Преобразование списка чисел
val numbers = List(1, 2, 3)
val doubled = numbers.map(n => n * 2)
// doubled = List(2, 4, 6)
```

Преимущества:
- Понятная задача
- Говорящие имена
- Простая концепция

### Рекомендации по количеству

**Для урока начального уровня:**
- 3-4 примера
- Базовый → Чуть сложнее → Практический

**Для урока среднего уровня:**
- 4-5 примеров
- Базовый → Усложнение → Комбинирование → Реальный use case

**Для урока продвинутого уровня:**
- 3-4 примера
- Все примеры близки к реальности
- Акцент на нюансы и edge cases

### Формат комментариев

```scala
// ✅ Хорошо: объясняет "почему"
// Используем lazy val для отложенной инициализации
lazy val config = loadConfiguration()

// ❌ Плохо: объясняет "что" (и так видно)
// Объявляем переменную x
val x = 5
```

## Примеры использования

В промпте для создания урока:
```
Создай 4 примера кода:
1. Базовый пример (3-5 строк) - синтаксис
2. Пример с одной дополнительной возможностью
3. Комбинирование 2-3 возможностей
4. Практический пример из реальной жизни

Каждый пример:
- Должен компилироваться
- Включает комментарии для ключевых моментов
- Показывает результат выполнения
- Идет от простого к сложному
```
