# Команда индексации курса

Индексирует контент курса в директории `content/` и создаёт навигационные файлы в `indexes/`.

**ВАЖНО:** Эта команда автоматически запускает отдельного агента для выполнения индексации, чтобы не перегружать основной контекст.

---

## Инструкция для Claude Code

При вызове этой команды ты должен:

1. **Запустить агента-координатора** используя Task tool:
   - subagent_type: "general-purpose"
   - description: "Координация индексации курса"
   - prompt: "Используй промпт из prompts/commands/index-course-coordinator.md"

2. **Дождаться завершения** работы координатора

3. **Показать пользователю** итоговый отчёт

**Пример запуска:**

```
Запускаю агента-координатора для индексации курса...

[Task tool запускает координатора с промптом index-course-coordinator.md]

[Координатор планирует итерации и запускает агентов-обработчиков...]

[Показать результат координатора пользователю]
```

**Архитектура:**

```
index-course.md (команда)
    ↓
index-course-coordinator.md (координатор)
    ↓
    ├─→ index-iteration-processor.md (обработчик итерации 1: модули 1-2)
    ├─→ index-iteration-processor.md (обработчик итерации 2: модули 3-5)
    ├─→ index-iteration-processor.md (обработчик итерации 3: остальные)
    └─→ index-generator.md (генератор индексов)
```

---

**НЕ выполняй индексацию в текущем контексте!** Всегда используй агента.
