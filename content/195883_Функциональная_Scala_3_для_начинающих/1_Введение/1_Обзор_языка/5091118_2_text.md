# Основные компьютерные термины и понятия

## ![Изображение](https://ucarecdn.com/271772f4-1a4c-4f16-bdf9-e4bbe33b3b67/)

**Компьютерная программа** - (Computer program) набор правил и инструкций для компьютера в виде описания (кода), которые он должен применить для обработки и выдачи данных.

**Прикладное программное обеспечение** - (Application Software), представляет собой программы, предназначенные для решения конкретных задач пользователей. Оно отличается от системного программного обеспечения, которое предназначено для обеспечения работы компьютера и его аппаратных компонентов.

**Аппаратная платформа** - (Hardware Platform), это физическое оборудование, на котором выполняется программное обеспечение. Она включает в себя все физические компоненты компьютера или устройства, такие как центральный процессор (CPU), оперативная память (RAM), жесткий диск, периферийные устройства ввода/вывода, сетевые интерфейсы и другие.

**Операционная платформа** или операционная система - (Operating System Platform), это системное программное обеспечение, которое предоставляет связь между аппаратным обеспечением и прикладным программным обеспечением. Примеры включают в себя платформы операционных систем, таких как Windows, Linux, macOS и т.д.

**Платформа** - комплекс аппаратной и операционной платформ. Описывает основную среду, на которой работает программное обеспечение

**Виртуальная машина** - (Virtual Machine, VM),  это программное обеспечение, предназначенное для выполнения в ней других программ, в среде, изолированной от фактической платформы. Это позволяет запускать программы на различных платформах без необходимости приспосабливаться к ним.

**JVM** - (Java Virtual Machine), представляет собой виртуальные машины для разных платформ, выполняющие специальный псевдо-код, подготовленный на разных языках программирования (изначально на языке Java). При создании программ, в этом случае, нет необходимости учитывать особенности конкретной платформы. Программы выполняются на внутренней операционной системе JVM, которая управляет памятью, потоками и безопасностью программ.

**Компилятор **— это программное обеспечение, которое преобразует исходный код программы, написанный на высокоуровневом языке программирования, в машинный код или псевдо-код (байт-код) для выполнения на компьютере или другом вычислительном устройстве. Компилятор выполняет лексический, синтаксический, семантический анализ кода, выявляет ошибки, оптимизирует код.

**Этап компиляции **- это подготовка исходного кода к началу исполнения программы на компьютере.

**Рефакторинг программы **(кода)— это процесс улучшения программы без изменения её внешнего поведения. Целью рефакторинга является облегчение понимания кода, повышение его гибкости, читаемости, а также возможности его дальнейшего развития.

**Программный проект** - это комплекс мероприятий и исходный материал для создания программного обеспечения.

**Библиотека** — это набор готовых программных элементов, предназначенных для повторного использования в различных проектах. Библиотеки предоставляют возможность не писать код с нуля.

**Стиль программирования** (парадигма программирования) -  общая методология, набор принципов и шаблонов к написанию кода программы. Она определяет основные принципы организации компьютерных программ. Стили программирования предоставляют программистам инструменты и концепции для решения задач различного типа.
