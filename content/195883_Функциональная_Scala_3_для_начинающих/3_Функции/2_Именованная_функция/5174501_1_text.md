# Именованная функция

**Именованная функция** - это особая конструкция **выражения**, имеющего **имя**, и возможность задания **параметров**.

Именованная функция представляет собой **блок кода**, который может быть вызван для выполнения определенной функциональности и получения **значения**.

Именованная функция может быть определена в **любом месте** программного кода.

Синтаксис определения:

`def functionName(parameters): ReturnType = // тело функции`

где:

- **def** - ключевое слово определения метода,

- **functionName** - имя функции

- **parameters** - секция определения параметров,

- **ReturnType** - тип результата (значения) функции,

- **тело функции** - выражение функции.

## Название функции

Название функции должно по возможности максимально точно отражать её назначение. Желательно использовать небольшую длину имени, так как придется использовать это имя в выражениях неоднократно.

Название функции должно начинаться с **буквы или символа**. Пробел использовать в имени недопустимо. Желательно начинать название функции с маленькой буквы. Желательно использовать английский язык.

Принято, чтобы названием функции была **глагольная фраза**, но иногда используют и существительное, как традиционное короткое название. Также в качестве названия можно использовать набор символов, но такие названия очень специфичны и не всегда понятны.

Пример названий функций:

def calculateSum() = ???
def getMass() = ???
def min() = ???
def ++() = ???

## Параметры функции

Параметры функции определяются в круглых скобках, через запятую.

Синтаксис объявления:

`(paramName: ParamType, ...)`

где:

- **paramName** - имя параметра,

- **ParamType** - типа параметра

Имя параметра должно придерживаться правил **имени значения**. Название типа параметра должно соответствовать **определенному типу данных**. Имена параметров должны быть разными.

Пример:

`(param1: Int, param2: Int)`

Здесь определено два параметра с именами param1 и param2. Оба параметра типа `Int`

Входные параметры представляют заранее **неизвестные** значения. Но **типы** этих значений известны, и соответственно можно определить допустимые операции с этими значением в теле функции.

Пример:

`param1 + param2 // в теле функции можно сложить два целых числа`

Количество параметров в функции может быть **любым**. Параметры могут **отсутствовать** совсем.

Пример определения функции без параметров:

`def getOne(): Int = 1`

## Выходное значение

Выходное значение функции всегда **одно** и имеет **тип**. Тип выходного значения можно не указывать, он будет вычислен.

Пример:

`def getOne(): Int = 1`

здесь выходное значение функции имеет тип `Int`

Пример:

`def getOne() = 1`

здесь выходное значение функции имеет тип `Int`, но он не указан.

## Полный пример определения функции

`def add(param1: Int, param2: Int): Int = param1 + param2`

здесь:

- **def **- ключевое слово определения метода,

- **add** - название метода,

- в круглых скобках список определяемых параметров,

- **param1: Int** - параметр с типом Int,

- **param2: Int** - параметр с типом Int,

- **Int** - тип результата (значения) метода,

- **=** - начало описания тела метода

- **param1 + param2** тело метода заданное выражением.

## Конструкция не первого класса

Именованная функция **не является** конструкцией **первого **класса. Таким образом именованная функция **не может** быть представлена как данные, не имеет функционального типа. Однако её можно легко **конвертировать** в анонимную функцию, которая имеет такие возможности. Конвертация в другой вид функции в Scala 3 часто может происходит автоматически, в частности это происходит при передаче именованной функции как аргумента другой функции.
