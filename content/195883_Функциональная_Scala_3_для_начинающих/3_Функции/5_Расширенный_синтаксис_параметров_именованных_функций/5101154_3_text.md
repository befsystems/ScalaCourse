# Необязательные параметры

Для параметра функции можно **указать значение**, которое будет использовано, если аргумент для параметра **не будет задан**. Такие значения называются **значениями по умолчанию**.

При вызове функции, указание аргументов для всех параметров является **обязательным требованием**, но для параметров со значением по умолчанию - это **не обязательно**.

Пример:

`def add(param1: Int, param2: Int = 1): Int = param1 + param2`

здесь:

- **param1: Int** - обязательный параметр с типом Int,

- **param2: Int = 1** - необязательный параметр с типом Int с аргументом по умолчанию 1.

Вызов метода:

val res = add(2, 2) // заданы все аргументы метода, результат 4
val res2 = add(2) // задан только обязательный аргумент, второй параметр использует аргумент по умолчанию 1, результат 3

Подстановка аргументов в соответствующие параметры функции производиться последовательно, по порядку их указания. И поэтому, необязательные параметры следует располагать последними в списке параметров, в этом случае можно удобно указывать только обязательные параметры.

Необязательных параметров может присутствовать **неограниченное** количество. **Все** параметры в методе могут быть необязательными.

В случае, когда правильный порядок подстановки аргументов в необязательные параметры невозможно обеспечить, необходимо использовать **именованные аргументы.**

Пример:

def add(param1: Int = 1, param2: Int = 2): Int = param1 + param2
val res = add(2, 2) // все аргументы распределены в порядке по параметрам.
val res2 = add(2) // Задан только первый параметр, второй по умолчанию.
val res3 = add(param2 = 2) // Задан только второй параметр именованным аргументом. Первый параметр по умолчанию.
