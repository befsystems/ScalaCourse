# Не указывать имена параметров в определении функции

В анонимных функциях можно не указывать имя параметра в определении, если параметр **не используется** в теле функции. Вместо имени можно указать **подстановочный знак **`_. ` Тем самым можно не придумывать название для параметра и показать его неиспользование.

Например:

val fun: Char => Unit = char => println("Hello") // параметр char объявлен, но не используется в теле функции
val fun: Char => Unit = _ => println("Hello") // его можно заменить подстановочным знаком

Обратите внимание! Хотя параметр и не используется, обозначить его наличие все равно необходимо для определения функционального значения.

Если тип функционального значения не задан, то тип параметра необходимо указать явно.

`val fun = (_: Char) => println("Hello") // его можно заменить подстановочным знаком`

Зная тип параметра и тип выходного значения легко вычисляется тип функционального значения `Char => Unit`

 

Неиспользованных параметров у функции может быть несколько.

Пример:

`val fun: (Char, Int) => Unit = (_, _) => println("Hello") // у функции два параметра, и оба не используются`

 

Зачем создавать анонимную функцию с неиспользуемым параметром? Это связано с тем, что анонимная функция почти всегда используется как встраиваемая функция со строго определенным типом. И этот тип является общим для многих вариантов функции с фиксированным набором параметров.
