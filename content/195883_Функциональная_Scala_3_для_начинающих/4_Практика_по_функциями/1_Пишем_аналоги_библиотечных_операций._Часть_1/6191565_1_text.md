# Операции и функции

В математике **функции **и **операции **— это разные сущности. Функция — это отображение одного множества в другое, имеющее четко определенные правила. Операция же чаще относится к выполнению действия между элементами внутри множества, например, сложение или умножение чисел.

В языке Scala граница между этими понятиями размыта. Это связано с тем, что синтаксис создания **операции и именованной функции** в языке Scala **одинаковы**. Для этого используется ключевое слово `def`. Кроме того, в объектно-ориентированном стиле это и вовсе считается одной сущностью - **методом** типа (метод класса). Поэтому можно услышать, что **операцию **часто называют **функцией **и наоборот. Но синтаксис использования этих действий существенно различается.

Примеры:

1 * 1 // использование операции умножения двух чисел

*(1, 1) // использование функции умножения двух чисел

Можно заметить, что **операции **типа и **функции **в языке Scala выполняют похожие действия в программе. Они вычисляют некий результат, и у них указаны операнды для операции, или аргументы для функции.

**Операции **записываются в **инфиксной нотации**, то есть операция пишется между операндами: `1 * 1`. Если операция унарная, то в **префиксной нотации  **`-1 `(операция перед операндом).

**Функции** записываются в **функциональной нотации**, то есть название функции пишется перед аргументами, а аргументы заключены в скобки: `*(1, 1)`.

**Операция **- это действие привязанное к первому операнду. Поэтому операцию `1 * 1` можно также записать как `1.*(1), `что означает, что у первого операнда имеется операция умножения со вторым операндом. Почти вся стандартная библиотека состоит из разных типов данных и их операций. 

Можно сказать, что операция, это функция, в теле которой использует один неявный параметр, не указанный в списке параметров. Этот неявный параметр задан вне определения функции, но функция его видит и использует. Соответственно, такая функция является нечистой, и её применение возможно только при наличии уже созданного первого параметра.

В дальнейших уроках, мы рассмотрим принципы создания операций типов. Использование операций для типов с небольшим количеством операндов считается более удобным для чтения и написания, поэтому так популярны даже в функциональном стиле.

**Функция (чистая функция)**, является самостоятельной сущностью, и имеет все необходимые ей параметры в явном виде, перечисленные в списке параметров. Явность всех параметров удобна для анализа и понимания кода.  В стандартной библиотеке, функции можно увидеть в пакете `scala.math`. Функции `sin`, `cos`, `sqrt`, — это примеры настоящих математических функций.

Можно конвертировать операцию в анонимную функцию. Без первого операнда нет и операции, поэтому первый операнд жестко встраивается в функцию.

val func: Int => Int = 1 + // конвертация операции в функцию
val res: Int = func(1) // результат 2

Понимание отличий между функцией и операцией нам необходимы для практики по функциям, где мы будем превращать операции из стандартной библиотеки в альтернативные чистые функции.
