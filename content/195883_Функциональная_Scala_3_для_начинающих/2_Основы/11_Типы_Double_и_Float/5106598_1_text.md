# Тип Double

Тип `Double` является моделью вещественных чисел.

**Вещественные числа** в математике широко используются для представления **дробных** и **действительных** чисел. Они включают в себя как **рациональные** **числа **(которые могут быть представлены отношением двух целых чисел), так и **иррациональные** **числа **(например, π или \(\sqrt2\)).

Тип данных `Double` реализован в формате числа **с плавающей точкой** двойной точности. Он использует стандарт [IEEE 754](https://ru.wikipedia.org/wiki/IEEE_754-2008) для представления вещественных чисел. Этот формат предоставляет высокую точность и широкий диапазон значений. Точка (разделитель целой и дробной частей) может плавать, то есть она может быть размещена в разных позициях числа в зависимости от его величины. Это позволяет представлять как очень маленькие, так и очень большие числа.

**Максимальное значение** для типа `Double` можно получить, используя константу `Double.MaxValue` `1.7976931348623157E308`

Для получения **минимального значение**, константу `Double.MinValue``-1.7976931348623157E308`

Буква `E` в числе означает начало экспоненты (порядка) числа. Это означает что указанное до экспоненты число (мантисса) необходимо умножить на 10 в степени экспоненты. Пример: `1.23E2 = 123`

Еще интересная константа - **минимальное позитивное значение**: Double.MinPositiveValue `4.9E-324`. Она представляет самое маленькое положительное число (самое близкое к нулю), которое может предоставить данный тип.

У типа имеются особые значения из мира вещественных чисел:

`Double.NaN` - не число`(`Not-a-Number), получается при `0.0 / 0.0`

`Double.POSITIVE_INFINITY` - позитивная бесконечность, получается при` 1.0 / 0.0`

`Double.NEGATIVE_INFINITY` - негативная бесконечность, получается при `-1.0 / 0.0`

## Литералы типа Double

**Обычный:** `3.14` или `-3.14`

**Экспоненциальный:** `2.5e6` или `-2.5E6`или `-2.5E-6`

**Без дробной части:** `2D` или `2d`

## Особенности типа Double

Представление вещественных чисел реализовано в типе не с абсолютной точностью, а некоторой **усеченной точностью**, что приводит к некоторым особенностям получаемых значений.

**Потеря точности** значения или **потери значения** при выполнении арифметических операций:

Пример:

val x: Double = 1.0
val y: Double = 1.0e-150
val result: Double = x + y //result имеет значение 1.0, хотя ожидалось что-то большее чем 1

Значения **нельзя **всегда** **надежно **сравнить**:

Пример:

`val res = 0.3 + 0.6 == 0.9 // res будет иметь false, хотя должно быть true.`

при сложении `0.3 + 0.6` получили число `0.8999999999999999`, которое близко к `0.9`, но не равно ему.

Пример:

`val res = 0.1 + 0.2 - 0.3 == 0.0 // res будет иметь false, хотя должно быть true.`

при вычислении `0.1 + 0.2 - 0.3` получили очень маленькое число 5.551115123125783E-17, которое близко к `0`, но не равно ему.

**Переполнение** значения возникает когда вычисляемое значение выходит из диапазона допустимых значений типа:

Пример:

`val overflowedValue: Double = Double.MaxValue * 2.0 // overflowedValue имеет значение Infinity`

Тип `Double`, в основном используется для **научных **расчетов, где использование вещественных чисел с известной точностью вполне приемлемо.

Использование типа `Double` может быть **неподходящим** для **валютных** финансовых расчетов из-за потенциальной потери точности и накопления ошибок округления в арифметических операциях.
