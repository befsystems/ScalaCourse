# Тип Unit

Тип `Unit` — это примитивный тип, предназначенный для представления особого значения, которое **не несёт в себе никакой информации**. Его иногда описывают как «отсутствие конкретного значения».

Основное назначение `Unit` — обозначать ситуации, когда важен **сам факт** того, что операция выполнена, но **не важен результат** вычислений.

### Единственное значение

`Unit` имеет **единственное возможное значение** — литерал `()`.

Он внешне похож на пустой кортеж, что отражает его суть: отсутствие содержимого.

`val u: Unit = ()`

С ним можно работать как с любым другим значением, хотя это редко полезно:

`val str: String = ().toString // вернёт "()"`

### Когда это используется

Например, выражение может что-то сделать — отправить команду внешней системе, записать данные в файл, вывести текст на экран — но при этом само значение результата не имеет смысла.

В таком случае выражение возвращает значение `Unit`:

`val res: Unit = println("Hello") // печать в консоль, но не вычисление результата`

Вызов `logMessage("Hello")` вернёт значение `()`, но единственное, что нам важно — это то, что сообщение напечатано. Анализировать такое значение бессмысленно: оно содержит только факт завершения действия, но не данные.

### Аналоги в других языках

В большинстве императивных языков программирования типу `Unit` соответствует `void` в C, C++, Java.

Главное различие — в Scala `Unit` является полноценным типом со значением `()`, тогда как `void` в этих языках — это скорее отсутствие типа.

Так же не следует путать `()` и `null`.

- `()` — это значение типа `Unit`.

- `null` — это значение -** пустая ссылка**, которая используется для объектных типов (о ней будет речь в будущем).

### Особенности в функциональном стиле

В чисто функциональном программировании `Unit` использовать нежелательно, так как его наличие часто говорит о **функциональной нечистоте** — побочных эффектах вроде ввода-вывода или изменения состояния.

Тем не менее, `Unit` играет важную роль в языке Scala:

- он используется во многих встроенных конструкциях,

- помогает при интеграции с Java и другими языками

### Неочевидное приведение типов

В Scala значение **любого типа** можно привести к `Unit`:

val a: Unit = 1 + 1 // исходное значение Int
val b: Unit = "строка" // исходное значение String
val c: Unit = true // исходное значение Boolean

Хотя в иерархии типов это явно не указано, компилятор позволяет подобные преобразования — это особенность реализации, которая решает часть практических задач, но может скрывать ошибки.
