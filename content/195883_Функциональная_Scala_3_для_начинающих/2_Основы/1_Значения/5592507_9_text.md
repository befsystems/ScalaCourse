# Ленивое именованное значение

Создание именованного значения может быть **ленивым**. Это означает, что реальное создание значения произойдет только в **момент первого использования** значения в неком выражении, а не в строчке кода определения значения.

Для определения ленивого значения нужно дополнительно указать ключевое слово** lazy.**

`lazy val numName = // данные (фактическое значение или выражение)`

Если ленивое значение **не будет использовано** при вычислении, то оно и **не будет** создано.

Однажды созданное ленивое значение уже **не будет создано повторно**, при повторном использовании, также как и все другие значения.

Ленивые значения позволяют **отложить **свое создание, до самого последнего момента перед использованием, тем самым снижая нагрузку в момент запуска программы, избегая ненужных вычислений, а также разрывая взаимные зависимости при создании значений.

Пример:

lazy val numName = 1 + 2 + 3 // Формальное определение ленивого значения
val res = numName // использование ленивого значения, при котором оно реально будет определено (вычислено)

**Ленивость всех сущностей** - отличительная черта функционального стиля и математики. Однако в Scala ленивость используется только при необходимости.
