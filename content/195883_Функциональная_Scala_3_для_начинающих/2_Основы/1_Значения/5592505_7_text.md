# Именованные значения

**Именованное значение** скрывает свое фактические значения под уникальным **именем**. Это позволяет использовать такое значение как заранее **неизвестное, абстрактное значение** (математическая переменная) в операциях и **алгебраических **выражениях.

**Алгебраические **выражения — это математические конструкции, состоящие из чисел, переменных и операций.

Пример математического алгебраического выражения:

\(2x^2+3xy−7\)

где x и у являются математическими переменными, или **именованными значениями** в терминах Scala, и заранее неизвестны при создании выражения.

## Создание именованного значения

Фактическое значение в коде создается и используется одновременно. **Именованное значение **сначала нужно **создать отдельным действием**, а затем уже использовать.

Синтаксис создания (определения) именованного значения:

`val valueName: ValueType = // данные (фактическое значение или выражение)`

где:

- **val **- служебное слово (от value - значение),

- **=** - символ присвоения,

- **ValueType** - тип значения (необязательно),

- **данные **- фактическое значение или выражение.

Тип значения при определении можно не указывать. Тип будет выведен автоматически.

Пример создания именованного значения `num`, и задание его литералом:

`val num: Int = 1`

Пример создания именованного значения `expression`, и задание его выражением:

`val expression = 1 + 2 + 3`

Именованное значение, самый **часто используемый** вид значения.

Еще пример создания и использования именованных значений:

val x = 2 // создание именованного значения x
val y = 3 // создание именованного значения y
2 * x * y − 7 // использование именованных значений в выражении

Создание именованного значения также называется **определением** значения. Определение именованного значения подразумевает **создание уникального имени значения и привязку его к значению**.

## Имя значения

Имя значения должно по возможности максимально точно отражать смысл его назначения. Но желательно использовать небольшую длину имени, тем самым сокращая длину выражений в котором оно используется.

Имя за значением закрепляется **навсегда**, и не меняется при работе программы.

**Некоторые правила создания имен для значений:**

- Имя каждого значения должно быть уникальным.

- Имя значения должно начинаться с **буквы**.

- Очень желательно начинать имя с **маленькой** буквы, чтобы не путать с именами типов данных.

- **Пробел **использовать в имени **недопустимо**.

- Символ` _` не принято использовать в именах, кроме специальных случаев.

- Желательно использовать только **английский язык** в названиях. Это избавляет от многих недоразумений и казусов.

- Принято, чтобы именем значения было **существительное **с прилагательными. Причем, существительное должно быть последним, показывая сущность значения.

- Если имя состоит из нескольких слов, то первые буквы слов (кроме первого слова) нужно писать большими. Такой стиль написания называется **верблюжьей **нотацией (CamelCase).

Пример сложного имени:  `calculatedTotalSum`

**Имена в обратных кавычках**.

В Scala можно использовать зарезервированные слова и другие нестандартные имена, если заключить их в обратные кавычки. Например:

val `type` = "example"
val aType = `type`

val `имя с пробелом` = 42
val `2name` = "second name"

Однако это считается **нежелательным** в обычном коде. Такой способ именования применяют:




для совместимости со сторонними API, где уже есть такие имена;




для временных или демонстрационных целей;




в автогенерированном коде.


В обычной практике лучше выбирать нормальные идентификаторы без обратных кавычек.

 

**Искусство **создания понятных и точных имен, наверное, самое сложное действие во всем программировании. Оно напрямую касается создания правильных моделей предметной области проекта. Смысл имени безразличен компьютеру и компилятору, но критически важен для программистов (слабое звено?), особенно в большом проекте. Проблемное имя почти наверняка вызовет массу логических ошибок и переделок. Правила создания имен обычно согласуются на уровне проектной документации, а некоторые ключевые имена составляют словарь терминов проекта.

Относитесь к созданию имени с ответственностью. Не спешите при создании имени, подумайте, как будет выглядеть ваше решение для других разработчиков. Не отдавайте код со временными или понятными только вам именами. Учитывайте многозначность используемых имен. 

**Помните**, как вы назовете конструкцию в коде, так её и будут использовать другие программисты.
