# Конструкция sealed trait + case objects

Конструкция **`sealed trait + case objects`** является старой модификацией конструктора суммарного типа, и почти полным аналогом конструкции `**enum** . `Конструкция ещё часто использоваться проектах в силу привычки. 

Определение конструкции более многословно, чем `enum`.

Пример:

sealed trait Status

object Status:
case object Active extends Status
case object Inactive extends Status
case class Pending(time: Int) extends Status

 Ключевые элементы:

**`sealed trait Status`** — запечатанный trait, базовый тип для суммы. Ключевое слово `sealed` гарантирует, что все наследники определены в том же файле, что критично для полноты проверки паттерн-матчинга.

**`case object`** — одиночные значения.

**`case class`** — значения с полями.

Использование** **конструкции аналогично использованию конструкции  **`enum `**

В конструкции** нет** встроенный операций, как в конструкции **`enum`**

### Сопоставление с образцом (Pattern Matching):

def describe(status: Status): String = status match
case Status.Active => "Система работает"
case Status.Inactive => "Система остановлена"
case Status.Pending(t) => s"Ожидание запуска через $t сек."

Компилятор проверяет **исчерпываемость** (exhaustivity) — если мы пропустим один из вариантов, получим предупреждение на этапе компиляции. Вкючение проверки обеспечивает указангие ключевое слова **`sealed`**
