# Базовые типы данных в системе Scala 3

Эти типы данных являются **фундаментальными единицами** в системе типов Scala 3. На их основе строятся более сложные типы. 

## Примитивные типы

val byte: Byte = 127
val short: Short = 32767
val int: Int = 42
val long: Long = 42L
val float: Float = 3.14f
val double: Double = 3.14159
val boolean: Boolean = true
val char: Char = 'A'
val unit: Unit = () // Примитивный singleton type

## Одноэлементные типы (Singleton Types)

### Единичный тип (Unit Type)

**Unit** - это примитивный singleton type с **единственным **значением `**()**.`Логически означает **присутствие значения**. Играет роль **единичного элемента** в алгебраических типах данных.

Является подтипом AnyVal - находится в иерархии примитивных типов.

def sideEffect(): Unit = println("Hello")
val unitValue: Unit = ()

println(unitValue == ()) // true
println(unitValue.toString) // ()

Используется для выхода функций с побочными эффектами и в математике системы типов.

### Пустой тип (Null Type) 

**Null** - это singleton type с единственным значением `null`. Логически означает **отсутствие значения**. 

Является подтипом **AnyRef**, но НЕ является подтипом **AnyVal**.  

val nullValue: Null = null
val nullString: String = null // Null <: String

Null существует в Scala для совместимости с Java. Избегайте использования `null`.

### Объекты-одиночки (Singleton Objects)

**Singleton objects** - это уникальные значения с со своим уникальным типом. Соответственно, каждый тип имеет только одно значение. Предназначен для создания пользовательских типов с единственным значением. 

Синтаксис определения значения:

object ИмяЗначения // Определение и создание уникального значения ИмяЗначения

Доступ к **типу значения** осуществляется через `.type`

Примеры:

object One // Определение и создание уникального значения One
object Two // Определение и создание уникального значения Two
object Three // Определение и создание уникального значения Three

val x: One.type = One // One.type — это ТИП (singleton type)
val y = One // One — это ЗНАЧЕНИЕ (единственный экземпляр)

println(One == Two) // false
println(One == x) // true

Обратите внимание,  что `One, Two, Three `- это значения, а не типы, хотя имя указано с большой буквы. Это является исключением из правил наименований, связанное с Java соглашениями.  

Созданные  значения `One, Two, Three`являются полностью независимыми , ничем не связанными друг с другом , и каждое из значений имеет свой индивидуальный тип .

Используется для создания сложных типов.

### Литеральные типы (Literal Types)

**Literal types** представляют типы с единственным значением для литералов (элементов) примитивных типов или типа String. Имя типа полностью совпадает с его значением.

val code: 404 = 404 // Int
val id: 100L = 100L // Long
val pi: 3.14f = 3.14f // Float
val e: 2.718 = 2.718 // Double
val flag: true = true // Boolean
val grade: 'A' = 'A' // Char
val mode: "debug" = "debug" // String

Обеспечивает: 

- Типобезопасность на уровне значений: компилятор проверяет точное соответствие

- Самодокументируемость: тип явно показывает допустимые значения

- Отсутствие магических констант: вместо комментариев — строгие типы

Предназначен для создания сложных типов и более узкой типизации примитивных типов. 

## Специальные типы

### Тип самого верхнего уровня (Any Type)

**Any** - это корень иерархии всех типов Scala (Top Type), **супертип всех других типов** в системе.

val anything: Any = 42
val anything2: Any = "hello"
val anything3: Any = Tuple3(1, 2, 3)

### Тип верхнего уровня для объектных типов (AnyRef  Type)

**AnyRef** - это **супертип всех объектных типов** (ReferenceTopType), эквивалент java.lang.Object.

val ref1: AnyRef = "string"
val ref2: AnyRef = Tuple3(1, 2, 3)

// AnyRef не включает value types
// val ref4: AnyRef = 42 // Ошибка компиляции!

### Тип верхнего уровня для примитивных типов (AnyVal Type)

**AnyVal** - это **супертип всех примитивных типов** (ValueTopType), включая пользовательские **value classes**.

val value1: AnyVal = 42
val value2: AnyVal = 3.14
val value3: AnyVal = true

### Тип отсутствия возможностей (Nothing Type)

**Nothing** - это **крайне нижний тип** (Bottom Type) в иерархии типов Scala, **подтип всех других типов**. Это тип** **означает **отсутствие возможностей** - невозможно создать значение типа Nothing, невозможно нормально вернуться из функции с таким типом. Используется в математике системы типов.

У типа **нет** реальных **значений**.

`def crash(): Nothing = throw new Exception() // Невозможно нормально вернуться `
